---
title: "List Configs"
api: "GET /api/v1/config/{user_id}/list"
description: "List all configuration types for a user."
---

Returns a list of all configuration types available for your user, including metadata such as version number, name, description, and enabled status. Use the `prefix` parameter to filter by configuration category.

## Authentication

Requires JWT with `read` permission. Your user must match the `{user_id}` in the URL.

## Path Parameters

<ParamField path="user_id" type="string" required>
  Your user identifier.
</ParamField>

## Query Parameters

<ParamField query="prefix" type="string">
  Optional prefix filter to narrow results by category. Common prefixes:
  - `agent:` -- Agent configurations
  - `pipeline:` -- Pipeline configurations
</ParamField>

## Response

<ResponseField name="items" type="array">
  Array of configuration entries.
</ResponseField>

### Configuration Entry

| Field | Type | Description |
|-------|------|-------------|
| `config_type` | string | Configuration type identifier (e.g., `user`, `agent:support_agent`, `pipeline:daily_report`) |
| `name` | string | Human-readable name |
| `description` | string or null | Configuration description |
| `enabled` | boolean | Whether this configuration is active |
| `version_number` | integer | Current version number (0 indicates the config exists but has not been versioned yet) |
| `created_at` | string | ISO 8601 creation timestamp |

## Error Responses

| Status | Description |
|--------|-------------|
| `401` | Missing or invalid credentials |
| `403` | Insufficient permissions or user mismatch |

```python Python
import httpx

response = httpx.get(
    "https://api.deha.one/proxy/acme_corp/config/api/v1/config/acme_corp/list",
    headers={"Authorization": f"Bearer {jwt_token}"},
    params={"prefix": "agent:"}
)
configs = response.json()
```
