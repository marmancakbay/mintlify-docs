---
title: "Create Schedule"
api: "POST https://api.deha.one/scheduler/schedules"
description: "Create a new dynamic schedule at runtime."
---

Creates a new schedule that fires pipelines, agents, or connectors on a cron, interval, or one-shot date trigger. Supports conditional execution -- the schedule only fires when its condition is met. Dynamic schedules persist across service restarts and one-shot (date) schedules are automatically cleaned up after execution.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token. `Bearer {your_token}`
</ParamField>
<ParamField header="X-User-Id" type="string" required>
  Your user identifier.
</ParamField>

## Request Body

<ParamField body="user_id" type="string" required>
  Your user identifier.
</ParamField>
<ParamField body="name" type="string" required>
  Human-readable schedule name.
</ParamField>
<ParamField body="trigger_type" type="string" default="date">
  Trigger type: `cron` (recurring cron expression), `interval` (recurring interval in seconds), or `date` (one-shot ISO 8601 datetime).
</ParamField>
<ParamField body="trigger_value" type="string" required>
  Cron expression (e.g., `0 9 * * *`), interval in seconds (e.g., `3600`), or ISO 8601 datetime (e.g., `2026-03-16T14:00:00`).
</ParamField>
<ParamField body="timezone" type="string" default="UTC">
  IANA timezone name (e.g., `Europe/Istanbul`, `America/New_York`).
</ParamField>
<ParamField body="action_type" type="string" default="pipeline">
  Action to perform when triggered: `pipeline`, `agent`, or `connector`.
</ParamField>
<ParamField body="pipeline_ref" type="string">
  Pipeline reference to trigger. Required when `action_type` is `pipeline`.
</ParamField>
<ParamField body="agent_ref" type="string">
  Agent reference to trigger. Required when `action_type` is `agent`.
</ParamField>
<ParamField body="input_params" type="object" default="{}">
  Input parameters passed to the action when triggered.
</ParamField>
<ParamField body="payload" type="object" default="{}">
  Custom payload data included in the trigger event.
</ParamField>
<ParamField body="condition_type" type="string" default="always">
  Condition evaluated before firing: `always` (unconditional), `sql` (DuckDB query), `llm` (AI decision), `threshold` (numeric comparison), or `composite` (AND/OR/NOT combination).
</ParamField>
<ParamField body="condition_query" type="string">
  SQL query or LLM prompt for condition evaluation. Required when `condition_type` is `sql`, `llm`, or `threshold`.
</ParamField>

## Condition Types

| Type | Description | Example |
|------|-------------|---------|
| `always` | Always fires (default) | -- |
| `sql` | Fires when SQL query returns truthy result | `SELECT COUNT(*) > 0 FROM orders WHERE status = 'pending'` |
| `llm` | AI evaluates prompt and returns true/false | `Based on sales data, should we trigger restocking?` |
| `threshold` | Fires when SQL result meets numeric threshold | Query + `threshold_value: 500`, `threshold_op: "gt"` |
| `composite` | Combines multiple conditions with AND/OR/NOT | Nested condition array |

## Response

<ResponseField name="success" type="boolean">
  Whether the schedule was created successfully.
</ResponseField>
<ResponseField name="data.schedule_id" type="string">
  Unique identifier for the created schedule.
</ResponseField>
<ResponseField name="data.name" type="string">
  The schedule name.
</ResponseField>
<ResponseField name="data.trigger_type" type="string">
  The trigger type.
</ResponseField>
<ResponseField name="data.trigger_value" type="string">
  The trigger value.
</ResponseField>
<ResponseField name="data.message" type="string">
  Confirmation message.
</ResponseField>

## Credit Cost

Each schedule trigger consumes **1 credit**.

## Error Responses

| Status | Description |
|--------|-------------|
| 400 | Invalid trigger configuration or cron expression |
| 401 | Missing or invalid authentication |
| 500 | Schedule creation error |
