---
title: "Rollback Config"
api: "POST /api/v1/config/{user_id}/{config_type}/rollback/{version_id}"
description: "Roll back a configuration to a previous version."
---

Rolls back a configuration to a previous version by creating a new version with the content of the target version. The full version history is preserved -- rollback does not delete any previous versions.

After rollback, you still need to [deploy](/api-reference/config-deploy) to make the rolled-back configuration active.

## Authentication

Requires JWT with `write` permission. Your user must match the `{user_id}` in the URL.

## Path Parameters

<ParamField path="user_id" type="string" required>
  Your user identifier.
</ParamField>
<ParamField path="config_type" type="string" required>
  The configuration type to roll back (e.g., `user`, `agent:support_agent`).
</ParamField>
<ParamField path="version_id" type="string" required>
  The target version identifier (UUID) to roll back to. This is the version whose content will be restored.
</ParamField>

## Response

Returns the newly created version (with the restored content). See [Get Config](/api-reference/config-get) for the full response schema.

The new version will have:
- A new `version_id` and incremented `version_number`
- The `yaml_content` from the target version
- A `commit_message` indicating this was a rollback operation

## Error Responses

| Status | Description |
|--------|-------------|
| `401` | Missing or invalid credentials |
| `403` | Insufficient permissions or user mismatch |
| `404` | Target version not found |
| `409` | Write lock could not be acquired (another write is in progress). Retry after a few seconds. |

<CodeGroup>
```bash cURL
curl -X POST "https://api.deha.one/proxy/acme_corp/config/api/v1/config/acme_corp/user/rollback/a1b2c3d4-e5f6-7890-abcd-ef1234567890" \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiIs..."
```

```python Python
import httpx

target_version_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"

response = httpx.post(
    f"https://api.deha.one/proxy/acme_corp/config/api/v1/config/acme_corp/user/rollback/{target_version_id}",
    headers={"Authorization": f"Bearer {jwt_token}"}
)
new_version = response.json()
print(f"Rolled back to version {new_version['version_number']}")
```


<Warning>
Remember to deploy after rolling back. The rollback only creates a new version in the version store -- it does not automatically activate the configuration. Call the [Deploy endpoint](/api-reference/config-deploy) to make the rolled-back configuration live.
</Warning>
