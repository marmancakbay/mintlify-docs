---
title: "HITL Respond"
api: "POST https://api.deha.one/api/v1/hitl/{workflow_id}/respond"
description: "Respond to a pending Human-in-the-Loop approval request."
---

Respond to a pending HITL (Human-in-the-Loop) workflow. The decision determines whether the AI-generated response is approved, rejected, revised, or the session is taken over by a human operator.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token. `Bearer {your_token}`
</ParamField>
<ParamField header="X-User-Id" type="string" required>
  Your user identifier. Automatically set by the gateway from your JWT.
</ParamField>
<ParamField header="X-User-Id" type="string" required>
  Staff user identifier. Automatically set by the gateway from your JWT.
</ParamField>

## Path Parameters

<ParamField path="workflow_id" type="string" required>
  The HITL workflow identifier.
</ParamField>

## Request Body

<ParamField body="decision" type="string" required>
  Decision: `approve` (accept AI response), `reject` (discard AI response), `revise` (provide alternative), or `takeover` (human takes control of session).
</ParamField>
<ParamField body="message" type="string">
  Optional message from the staff member.
</ParamField>
<ParamField body="content" type="string">
  Optional replacement text for the AI response. Used with `revise` decision.
</ParamField>

## Response

<ResponseField name="status" type="string">
  Operation status (`ok`).
</ResponseField>
<ResponseField name="workflow_id" type="string">
  The workflow identifier.
</ResponseField>
<ResponseField name="decision" type="string">
  The decision that was made.
</ResponseField>

## Error Responses

| Status | Description |
|--------|-------------|
| 401 | Missing authentication headers |
| 403 | Workflow belongs to a different user |
| 404 | Workflow not found |
| 409 | Workflow is no longer pending (already resolved) |

## Session Takeover

`POST https://api.deha.one/api/v1/hitl/{workflow_id}/takeover`

Takes over a session from AI control to human control. The session is locked to the staff member.

### Request Body

| Field | Type | Description |
|-------|------|-------------|
| `reason` | string | Optional reason for the takeover |

---

## Release Session

`POST https://api.deha.one/api/v1/hitl/{workflow_id}/release`

Releases a session back to AI control after a human takeover.
