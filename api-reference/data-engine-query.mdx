---
title: "Query Data"
api: "POST https://api.deha.one/data/query"
description: "Execute a SQL query against your user's data warehouse."
---

Execute a SQL query against your user's isolated data warehouse and return the results. Only read operations and safe write operations are permitted through a security sandbox.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token. `Bearer {your_token}`
</ParamField>
<ParamField header="X-User-Id" type="string" required>
  Your user identifier.
</ParamField>

## Request Body

<ParamField body="user_id" type="string" required>
  Your user identifier.
</ParamField>
<ParamField body="sql" type="string" required>
  SQL query to execute.
</ParamField>
<ParamField body="max_rows" type="integer" default="10000">
  Maximum rows to return. Maximum allowed value is 100,000.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the query executed successfully.
</ResponseField>
<ResponseField name="data.status" type="string">
  Query status: `success` or `error`.
</ResponseField>
<ResponseField name="data.records" type="array">
  Array of result rows as JSON objects.
</ResponseField>
<ResponseField name="data.record_count" type="integer">
  Number of rows returned.
</ResponseField>
<ResponseField name="data.columns" type="array">
  List of column names in the result set.
</ResponseField>
<ResponseField name="data.duration_ms" type="number">
  Query execution time in milliseconds.
</ResponseField>
<ResponseField name="error" type="string">
  Error message if the query failed. `null` on success.
</ResponseField>

<Note>
**SQL Sandbox:** For security, the following statements are blocked: `DROP DATABASE`, `ALTER SYSTEM`, `ATTACH`, `DETACH`, `COPY TO`, `EXPORT`, and `IMPORT`. Allowed statements include `SELECT`, `INSERT INTO`, `CREATE TABLE`, `CREATE OR REPLACE`, and `WITH`.
</Note>

## Error Responses

| Status | Description |
|--------|-------------|
| 400 | Invalid SQL or blocked statement |
| 401 | Missing or invalid authentication |
| 403 | User access denied |
| 500 | Internal query execution error |

<CodeGroup>
```bash cURL
curl -X POST "https://api.deha.one/data/query" \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiIs..." \
  -H "X-User-Id: acme_corp" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "acme_corp",
    "sql": "SELECT * FROM customers LIMIT 10"
  }'
```

```python Python
import httpx

response = httpx.post(
    "https://api.deha.one/data/query",
    headers={
        "Authorization": f"Bearer {jwt_token}",
        "X-User-Id": "acme_corp"
    },
    json={
        "user_id": "acme_corp",
        "sql": "SELECT * FROM customers LIMIT 10"
    }
)
```

