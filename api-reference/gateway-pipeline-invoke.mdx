---
title: "Invoke Pipeline"
api: "POST /api/v1/agent/{user_id}/pipeline/{pipeline_name}/invoke"
description: "Synchronously invoke a multi-step pipeline and receive the result."
---

Invokes a named pipeline and returns the result synchronously. Pipelines orchestrate multi-step workflows that can include data processing, multiple agent calls, transformations, and integrations.

The request has a **65-second timeout**. For pipelines that may take longer, consider breaking them into smaller steps or using asynchronous patterns.

## Authentication

Requires an API key in the `Authorization` header with the `Bearer` prefix. The key's user must match the URL `{user_id}`.

## Path Parameters

<ParamField path="user_id" type="string" required>
  Your user identifier.
</ParamField>
<ParamField path="pipeline_name" type="string" required>
  The name of the pipeline to invoke (as defined in your pipeline configuration).
</ParamField>

## Request Body

<ParamField body="message" type="string" required>
  The input message or trigger for the pipeline.
</ParamField>
<ParamField body="session_id" type="string">
  Optional session identifier. Auto-generated if omitted.
</ParamField>
<ParamField body="context" type="object">
  Optional key-value context passed to the pipeline. Available to all steps during execution.
</ParamField>

## Response

<ResponseField name="reply" type="string">
  The pipeline's output text.
</ResponseField>
<ResponseField name="session_id" type="string">
  The session identifier used for this execution.
</ResponseField>
<ResponseField name="trace_id" type="string">
  Trace identifier for debugging and support inquiries.
</ResponseField>
<ResponseField name="pipeline_name" type="string">
  The name of the pipeline that was executed.
</ResponseField>
<ResponseField name="model" type="string">
  The LLM model used (if applicable).
</ResponseField>

## Error Responses

| Status | Description |
|--------|-------------|
| `401` | Invalid or missing API key |
| `403` | API key user does not match URL user |
| `404` | Pipeline not found |
| `429` | Rate limit exceeded |
| `504` | Pipeline did not complete within the 65-second timeout |
