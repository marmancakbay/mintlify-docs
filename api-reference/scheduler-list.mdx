---
title: "List Schedules"
api: "GET https://api.deha.one/scheduler/schedules"
description: "List active schedule configurations for your user."
---

Returns all active schedule configurations for your user, including both YAML-defined and dynamically created schedules. Each schedule shows its trigger type, timing, enabled state, and associated action.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token. `Bearer {your_token}`
</ParamField>
<ParamField header="X-User-Id" type="string" required>
  Your user identifier.
</ParamField>

## Query Parameters

<ParamField query="user_id" type="string" required>
  Your user identifier.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the request completed successfully.
</ResponseField>
<ResponseField name="data.schedules" type="array">
  Array of schedule configuration objects.
</ResponseField>
<ResponseField name="data.count" type="integer">
  Total number of active schedules.
</ResponseField>

## Schedule Object

| Field | Type | Description |
|-------|------|-------------|
| `schedule_id` | string | Unique schedule identifier |
| `name` | string | Human-readable name |
| `trigger_type` | string | `cron`, `interval`, or `date` |
| `trigger_value` | string | Cron expression, interval seconds, or ISO datetime |
| `timezone` | string | IANA timezone |
| `enabled` | boolean | Whether the schedule is active |
| `action` | object | Action configuration with `action_type`, `pipeline_ref`, `agent_ref`, `connector_ref`, and `input_params` |

## Error Responses

| Status | Description |
|--------|-------------|
| 401 | Missing or invalid authentication |
