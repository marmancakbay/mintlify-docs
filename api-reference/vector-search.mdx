---
title: "Semantic Search"
api: "POST /api/v1/search"
description: "Search the knowledge base using natural language queries."
---

Performs a semantic search against your user's knowledge base using cosine similarity. The query text is embedded using your configured model and compared against stored document vectors to find the most relevant results.

This is a synchronous endpoint that returns results immediately.

## Authentication

Requires JWT authentication via the proxy endpoint.

## Request Body

<ParamField body="user_id" type="string" required>
  Your user identifier.
</ParamField>
<ParamField body="query_text" type="string" required>
  Natural language search query. The query is embedded and compared against your stored documents.
</ParamField>
<ParamField body="limit" type="integer" default="5">
  Maximum number of results to return (1-100).
</ParamField>
<ParamField body="score_threshold" type="number" default="0.7">
  Minimum similarity score (0.0 to 1.0). Results below this threshold are excluded. Lower values return more results but with less relevance.
</ParamField>
<ParamField body="collection_name" type="string">
  Target collection name. If omitted, searches the default collection configured for your user (`{user_id}_knowledge`).
</ParamField>
<ParamField body="filters" type="object">
  Key-value metadata filters to narrow results. Only documents with matching metadata are returned. Example: `{"department": "legal", "language": "en"}`.
</ParamField>
<ParamField body="embedding_config" type="object">
  Override the embedding model for this query. If omitted, uses your user's configured embedding model.
  - `provider` (string) -- Embedding provider: `openai`, `gemini`, `huggingface`
  - `model_name` (string) -- Model identifier
  - `dimension` (integer) -- Vector dimension
</ParamField>

## Response

<ResponseField name="results" type="array">
  Array of search results, ordered by relevance (highest score first). Each result contains:
  - `id` (string) -- Unique vector identifier
  - `content` (string) -- The matched text chunk
  - `score` (number) -- Similarity score (0.0 to 1.0, higher is more relevant)
  - `metadata` (object) -- Document metadata including `filename`, `chunk_index`, `user_id`, `ingested_at`, and any custom metadata
</ResponseField>
<ResponseField name="count" type="integer">
  Number of results returned.
</ResponseField>
<ResponseField name="processing_time" type="number">
  Search duration in seconds.
</ResponseField>

## Error Responses

| Status | Description |
|--------|-------------|
| `401` | Missing or invalid credentials |
| `500` | Search failed (e.g., embedding model error or database unavailable) |

<CodeGroup>
```bash cURL
curl -X POST "https://api.deha.one/proxy/acme_corp/vector/api/v1/search" \
  -H "Authorization: Bearer eyJhbGciOiJSUzI1NiIs..." \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "acme_corp",
    "query_text": "What is the return policy?",
    "limit": 5,
    "score_threshold": 0.7,
    "filters": {"department": "legal"}
  }'
```

```python Python
import httpx

response = httpx.post(
    "https://api.deha.one/proxy/acme_corp/vector/api/v1/search",
    headers={"Authorization": f"Bearer {jwt_token}"},
    json={
        "user_id": "acme_corp",
        "query_text": "What is the return policy?",
        "limit": 5,
        "score_threshold": 0.7,
        "filters": {"department": "legal"}
    }
)
results = response.json()

for result in results["results"]:
    print(f"[{result['score']:.2f}] {result['content'][:100]}...")
```

<Tip>
Adjust the `score_threshold` based on your use case. For precise answers (FAQ lookup), use a higher threshold (0.8-0.9). For exploratory search, use a lower threshold (0.5-0.7) to surface more potentially relevant results.
</Tip>
