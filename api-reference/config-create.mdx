---
title: "Create Config Version"
api: "POST /api/v1/config/{user_id}/{config_type}"
description: "Create a new version of a configuration."
---

Creates a new version of the specified configuration type. The previous current version is automatically archived. A diff from the previous version is computed and stored automatically.

If the submitted content is identical to the current version (same SHA-256 hash), the existing version is returned without creating a duplicate.

## Authentication

Requires JWT with `write` permission. Your user must match the `{user_id}` in the URL.

## Path Parameters

<ParamField path="user_id" type="string" required>
  Your user identifier.
</ParamField>
<ParamField path="config_type" type="string" required>
  The configuration type to create a version for. Examples:
  - `user` -- Main user configuration
  - `channels` -- Channel configurations
  - `agent:support_agent` -- A specific agent configuration
  - `pipeline:daily_report` -- A specific pipeline configuration
  - `data` -- Data refinery configuration
</ParamField>

## Request Body

<ParamField body="yaml_content" type="string" required>
  The full YAML configuration content. Must be valid YAML.
</ParamField>
<ParamField body="commit_message" type="string">
  Optional description of the change. Recommended for audit trail and team collaboration.
</ParamField>

## Response (201 Created)

Returns the newly created version. See [Get Config](/api-reference/config-get) for the full response schema.

<ResponseField name="version_id" type="string">
  Unique identifier for the new version.
</ResponseField>
<ResponseField name="user_id" type="string">
  User identifier.
</ResponseField>
<ResponseField name="config_type" type="string">
  Configuration type.
</ResponseField>
<ResponseField name="version_number" type="integer">
  The new version number.
</ResponseField>
<ResponseField name="content_hash" type="string">
  SHA-256 hash of the YAML content.
</ResponseField>
<ResponseField name="yaml_content" type="string">
  The submitted YAML content.
</ResponseField>
<ResponseField name="diff_from_previous" type="string">
  Unified diff from the previous version.
</ResponseField>
<ResponseField name="created_by" type="string">
  Your user ID.
</ResponseField>
<ResponseField name="created_at" type="string">
  ISO 8601 timestamp.
</ResponseField>
<ResponseField name="commit_message" type="string">
  The commit message you provided.
</ResponseField>
<ResponseField name="parent_version_id" type="string">
  Version ID of the previous version.
</ResponseField>
<ResponseField name="status" type="string">
  Always `current` for newly created versions.
</ResponseField>

## Error Responses

| Status | Description |
|--------|-------------|
| `401` | Missing or invalid credentials |
| `403` | Insufficient permissions or user mismatch |
| `409` | Write lock could not be acquired (another write is in progress). Retry after a few seconds. |

```python Python
import httpx

yaml_content = """
name: support_agent
type: conversation
model: gpt-4o
nodes:
  - id: llm_node
    type: llm
    config:
      system_prompt: "You are a helpful support agent."
""".strip()

response = httpx.post(
    "https://api.deha.one/proxy/acme_corp/config/api/v1/config/acme_corp/agent:support_agent",
    headers={"Authorization": f"Bearer {jwt_token}"},
    json={
        "yaml_content": yaml_content,
        "commit_message": "Updated system prompt for better tone"
    }
)
```
