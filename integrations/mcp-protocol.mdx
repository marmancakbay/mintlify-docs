DEHA ONE supports the **Model Context Protocol (MCP)** -- an open standard for connecting AI agents to external tools. If a tool provider offers an MCP server, you can connect it to your agents and start using it right away.

---

## What is MCP?

The Model Context Protocol is a standard way for AI applications to discover and use external tools. Instead of building a custom integration for each tool, MCP provides a universal interface:

- **Tool discovery** -- Your agent automatically learns what tools are available on the connected server
- **Standardized calling** -- Tools are called in a consistent way, regardless of who built them
- **Typed parameters** -- Each tool declares what parameters it accepts, so your agent knows exactly what to provide

MCP is an open standard supported by a growing ecosystem of tool providers.

---

## Connecting an MCP server

To connect an MCP-compatible tool server, add it to your integration configuration:

```yaml
mcp_servers:
  slack:
    url: https://mcp.slack.com/rpc
    enabled: true
    namespace: slack
    auth:
      type: bearer
      secret_ref: SLACK_MCP_TOKEN
```

Once connected, DEHA ONE automatically discovers all available tools from the server and makes them available to your agents. For example, connecting a Slack MCP server might give your agents tools like `slack__send_message`, `slack__list_channels`, and `slack__search_messages`.

---

## Filtering available tools

You can control which tools from an MCP server are available to your agents:

```yaml
mcp_servers:
  github:
    url: https://mcp.github.com/rpc
    namespace: github
    allowed_tools:
      - list_repos
      - create_issue
      - get_pull_request
    blocked_tools:
      - delete_repo
```

- Use **allowed_tools** to expose only specific tools (whitelist)
- Use **blocked_tools** to hide specific tools (blacklist)
- Leave both empty to expose all tools from the server

## Authentication

MCP servers support the same authentication methods as REST API tools:

| Method | Configuration |
|--------|--------------|
| **Bearer Token** | `type: bearer` with `secret_ref` |
| **API Key** | `type: api_key` with `secret_ref` and `header_key` |
| **OAuth2** | `type: oauth2` with token URL and credentials |
| **Basic Auth** | `type: basic` with username and password references |

All credentials are stored securely in your encrypted vault.

---

## DEHA ONE as an MCP server

DEHA ONE can also expose your agents as MCP servers. This means external systems that support MCP can connect to your agents and use them as tools. This is useful for:

- Connecting DEHA ONE agents to other AI platforms
- Allowing external systems to invoke your agents programmatically
- Agent-to-agent communication across different platforms

---

## Automatic tool refresh

When you add, remove, or modify MCP server configurations, DEHA ONE automatically refreshes the available tools. You don't need to restart anything -- changes take effect within seconds.
