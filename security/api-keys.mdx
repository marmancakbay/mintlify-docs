API keys let you connect external applications, automation tools, and backend services to DEHA ONE without requiring a user login. Use them for server-to-server integrations, tools like n8n or Zapier, or any custom application that needs to interact with your agents programmatically.

---

## Creating an API key

You can create API keys from the DEHA ONE dashboard:

1. Go to **Settings** > **API Keys**
2. Click **Create New Key**
3. Give the key a descriptive label (e.g., "Zapier Production", "CRM Integration")
4. Copy the key immediately -- it will only be shown once

<Warning>
  Your API key is displayed only at the moment of creation. Copy it and store it securely right away. If you lose it, you will need to create a new one.
</Warning>

---

## Using your API key

Include the API key in the `Authorization` header of your requests:

<CodeGroup>
```bash curl
curl -X POST https://api.deha.one/api/v1/agent/{user_id}/{agent_name}/invoke \
  -H "Authorization: Bearer dh_live_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What is the status of order #12345?",
    "user_id": "system-integration",
    "session_id": "integration-session-001"
  }'
```

```python Python
import httpx

response = httpx.post(
    "https://api.deha.one/api/v1/agent/your_user/support_agent/invoke",
    headers={
        "Authorization": "Bearer dh_live_your_api_key_here",
        "Content-Type": "application/json"
    },
    json={
        "message": "What is the status of order #12345?",
        "user_id": "system-integration",
        "session_id": "integration-session-001"
    },
    timeout=65.0
)

print(response.json())
```

```javascript Node.js
const response = await fetch(
  "https://api.deha.one/api/v1/agent/your_user/support_agent/invoke",
  {
    method: "POST",
    headers: {
      "Authorization": "Bearer dh_live_your_api_key_here",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      message: "What is the status of order #12345?",
      user_id: "system-integration",
      session_id: "integration-session-001"
    })
  }
);

const data = await response.json();
console.log(data);
```
</CodeGroup>

## What you can do with API keys

API keys give you access to the following actions:

| Action | Description |
|--------|-------------|
| **Invoke an agent** | Send a message to any of your agents and get a response |
| **Run a pipeline** | Trigger an automated pipeline |
| **MCP endpoint** | Connect via the Model Context Protocol |

## Managing your keys

### View all keys

Go to **Settings** > **API Keys** to see all your active keys, including their labels and when they were last used.

### Delete a key

Click the delete button next to any key to revoke it immediately. Any integrations using that key will stop working right away.

### Rotate a key

To rotate a key safely:

1. Create a new key
2. Update your integration to use the new key
3. Verify the integration works with the new key
4. Delete the old key

<Info>
  Each organization can have up to **5 active API keys**. Delete unused keys before creating new ones if you have reached the limit.
</Info>

## Best practices

<Tip>
  Use a descriptive label when creating each key so you can easily tell which integration it belongs to.
</Tip>

- **One key per integration.** Create a separate key for each external system. This way, you can rotate or revoke access for one integration without affecting others.
- **Rotate keys regularly.** Even if a key has not been compromised, periodic rotation limits the impact of a potential leak.
- **Store keys securely.** Never hardcode API keys in source code. Use your application's environment variables or secret management system.
- **Don't share keys.** Each team member or system should have its own key when possible.
- **Delete unused keys.** If an integration is no longer active, delete its key to reduce your attack surface.
