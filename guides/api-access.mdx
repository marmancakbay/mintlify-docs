## What You Will Set Up

Create API keys so you can interact with DEHA ONE programmatically -- send messages to agents, trigger automations, query data, and more from your own applications.

<Info>
Before you begin, you need:
- A DEHA ONE account with admin access to your workspace
</Info>

---

<Steps>

### Generate an API Key

1. In the dashboard, go to **Settings > API Keys**.
2. Click **Create API Key**.
3. Give it a descriptive name (e.g., "Production Backend" or "Reporting Script").
4. Choose a role:
   - **Admin** -- Full access to all features
   - **User** -- Standard access (send messages, query data)
5. Set an expiration (optional but recommended for security).
6. Click **Create**.

<Warning>
The API key is shown only once when created. Copy it immediately and store it securely. If you lose it, you will need to generate a new one.
</Warning>

### Use Your API Key

Include your API key in the `X-API-Key` header for all requests:

<CodeGroup>

```bash cURL
curl https://api.deha.one/api/v1/agents/your_workspace/support_agent/invoke \
  -H "X-API-Key: deha_key_abc123..." \
  -H "Content-Type: application/json" \
  -d '{"message": "Hello, I need help."}'
```

```python Python
import httpx

client = httpx.Client(
    base_url="https://api.deha.one",
    headers={"X-API-Key": "deha_key_abc123..."}
)

# Talk to an agent
resp = client.post(
    "/api/v1/agents/your_workspace/support_agent/invoke",
    json={"message": "Hello, I need help."}
)
print(resp.json())
```

```javascript Node.js
const response = await fetch(
  "https://api.deha.one/api/v1/agents/your_workspace/support_agent/invoke",
  {
    method: "POST",
    headers: {
      "X-API-Key": "deha_key_abc123...",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ message: "Hello, I need help." })
  }
);
const data = await response.json();
```

</CodeGroup>

### Common API Operations

Here are the most common things you can do with the API:

**Talk to an agent:**
```bash
curl -X POST https://api.deha.one/api/v1/agents/{workspace}/{agent}/invoke \
  -H "X-API-Key: your_key" \
  -d '{"message": "Your question here"}'
```

**Query your data:**
```bash
curl -X POST https://api.deha.one/query/ask \
  -H "X-API-Key: your_key" \
  -d '{"question": "How many active users do we have?"}'
```

**Upload a document to the knowledge base:**
```bash
curl -X POST https://api.deha.one/vector/ingest \
  -H "X-API-Key: your_key" \
  -F "file=@document.pdf" \
  -F "collection_name=my_knowledge"
```

**Trigger an automation:**
```bash
curl -X POST https://api.deha.one/api/v1/pipelines/invoke \
  -H "X-API-Key: your_key" \
  -d '{"pipeline_id": "daily_sync", "parameters": {}}'
```

### Understand Rate Limits

API requests are subject to rate limits. The response headers tell you where you stand:

| Header | What It Means |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests allowed per window |
| `X-RateLimit-Remaining` | How many requests you have left |
| `X-RateLimit-Reset` | When the limit resets (Unix timestamp) |
| `Retry-After` | How long to wait if you hit the limit (seconds) |

### Manage Your Keys

**View all keys:** Go to **Settings > API Keys** in the dashboard.

**Revoke a key:** Click the delete icon next to any key. The key stops working immediately.

**Rotate keys:** Create a new key, update your applications, then revoke the old key.

<Check>
Test your API key by making a simple request to the health endpoint: `curl https://api.deha.one/health -H "X-API-Key: your_key"`.
</Check>

</Steps>

## Security Best Practices

- **Never expose API keys in client-side code** (browser JavaScript, mobile apps).
- **Rotate keys periodically** -- set expiration dates and generate new keys before they expire.
- **Use the minimum required role** for each key.
- **Store keys in environment variables or a secret manager**, never in source code.
- **Revoke unused keys** promptly.
