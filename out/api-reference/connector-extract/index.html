      Extract Data

      Extract data from a configured external data source.

      POST
      /
      connectors
      /
      extract
      Extract Data
      curl --request POST \
  --url https://api.deha.one/connectors/extract \
  --header 'Authorization: <authorization>' \
  --header 'Content-Type: application/json' \
  --header 'X-User-Id: <x-user-id>' \
  --data '
{
  "user_id": "<string>",
  "connector_name": "<string>",
  "target_entity": "<string>",
  "filters": {},
  "limit": 123
}
'
      {
  "success": true,
  "data.status": "<string>",
  "data.records": [
    {}
  ],
  "data.record_count": 123,
  "data.cursor": "<string>",
  "data.has_more": true,
  "data.connector_type": "<string>",
  "data.sync_mode": "<string>",
  "data.duration_ms": 123,
  "error": "<string>"
}
      Triggers a data extraction operation from a configured connector. Supports REST APIs, databases (PostgreSQL, MySQL), files (CSV, Excel, JSON, Parquet), and webhooks. The connector uses incremental or full-refresh sync modes based on its configuration.

      ​
      Authentication

      ​
      Authorization
      string
      required
      Bearer token. Bearer {your_token}
      ​
      X-User-Id
      string
      required
      Your user identifier.

      ​
      Request Body

      ​
      user_id
      string
      required
      Your user identifier.
      ​
      connector_name
      string
      required
      Name of the connector as defined in your user configuration.
      ​
      target_entity
      string
      Target table or endpoint to extract from. Defaults to the connector’s configured default entity.
      ​
      filters
      object
      Key-value filter parameters passed to the connector for server-side filtering.
      ​
      limit
      integer
      Maximum records to extract. Maximum allowed value is 100,000.

      ​
      Response

      ​
      success
      boolean
      Whether the extraction completed successfully.
      ​
      data.status
      string
      Extraction status: success or error.
      ​
      data.records
      array
      Array of extracted data records as JSON objects.
      ​
      data.record_count
      integer
      Number of records extracted.
      ​
      data.cursor
      string
      Cursor value for the next incremental sync. Use this to track sync progress.
      ​
      data.has_more
      boolean
      Whether more data is available beyond the current extraction.
      ​
      data.connector_type
      string
      Type of connector used: rest_api, postgresql, mysql, file_storage, csv, excel, or webhook.
      ​
      data.sync_mode
      string
      Sync mode used: full_refresh, incremental, or cdc.
      ​
      data.duration_ms
      number
      Extraction duration in milliseconds.
      ​
      error
      string
      Error message if extraction failed. null on success.

      ​
      Sync Modes

      ModeDescription
      full_refreshExtracts all records on every run
      incrementalOnly extracts records newer than the last sync cursor
      cdcChange Data Capture: similar to incremental using a timestamp column

      ​
      Credit Cost

      Each extraction consumes 3 credits.

      ​
      Error Responses

      StatusDescription
      400Connector not found or invalid configuration
      401Missing or invalid authentication
      500Extraction execution error
